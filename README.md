# Setup instructions
click link below
you can begin using right away, for the ability to save words and create your own definitions you must sign up and register an account.
Once an account is made you can login and search words and then save them to your default folder.
You can create new folders and label them, you can then move any words you have saved into another folder you have created and will be saved there. 
For any word you can also create a new definition that will be shown alongside the dictionary definition.

# Reflection
I chose react for my frontend because in working on my capstone project I used a lot of react this semester making me more comfortable using it. For the backend I chose Node.js with express also becuase of familiarity. For the database I used postgresSQL to create 3 tables, users, folders, and saved_words. My biggest issue other than getting the css to work properly, was filtering words through each folder, I wanted each word to automatically save to the default folder and then the user to be able to add it to their own folders as they create them, that way there is an actual purpose to having more than just the default folder. I learned a lot about full stack development I think, though I did have some experience coming into the class I was able to hone my skills I already had while adding new skills and knowledge. I think now I have a better understanding of how each part of the different parts of the project connect from the database to the apis and the front and back ends. I would have added a word or docs like section where users could have all the tools they need to write with rather than just the ability to save words.
